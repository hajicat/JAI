# 2026-04-09

## 吉动盲盒项目（jlai-dating）

### 完成的工作

1. **依赖问题修复**
   - 原项目使用 `better-sqlite3`（需要 C++ 编译，Windows 无 Visual Studio 会失败，Vercel 无服务器函数也无法运行）
   - 替换为 `@libsql/client`（纯 JS，支持本地文件 SQLite 和 Turso 云端，无需编译）

2. **所有 API 路由改写**
   - 从同步 better-sqlite3 API 改为 async @libsql/client API
   - 涉及：login、register、me（GET/POST）、survey、match（GET/POST）、invite、admin/users、admin/codes、admin/match

3. **Bug 修复**
   - login API 缺少 `surveyCompleted` 字段（前端跳转逻辑依赖此字段）
   - globals.css 中 `@import` 顺序错误（必须在 `@tailwind` 之前）
   - globals.css 缺少 `@keyframes float` 定义（tailwind 的 `animate-float` 依赖它）
   - next.config.js 移除了 better-sqlite3 的 externalPackages

4. **新增文件**
   - `.env.local`：本地开发环境变量
   - `.env.example`：部署参考
   - `.gitignore`：忽略数据库文件和 .env
   - `DEPLOY.md`：详细的 Vercel + Turso 部署指南

5. **本地运行验证**
   - `npm run dev` → ✓ Ready in 8.9s，所有 API 200，无报错
